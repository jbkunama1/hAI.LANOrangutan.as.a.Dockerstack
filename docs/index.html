🦧 LAN Orangutan – Docker Stack

✅ Produktiv-ready Docker Portainer nmap LAN-Scanner Homelab

Produktiv-Setup für LAN Orangutan als Portainer-Stack im Heimnetz. Scannt dein LAN, erkennt Geräte, zeigt offene Ports und MAC-Adressen – alles über ein sauberes Web-Dashboard.

🚀 Schritt-für-Schritt

  1. Upstream-Repo klonen & Image bauen 
    git clone https://github.com/291-Group/LAN-Orangutan.git
cd LAN-Orangutan
docker build -t lan-orangutan:latest .
  2. Setup-Script ausführen 
    git clone https://github.com/jbkunama1/hAI.LANOrangutan.as.a.Dockerstack.git
cd hAI.LANOrangutan.as.a.Dockerstack
chmod +x setup.sh && sudo ./setup.sh
  3. Subnetz in Config eintragen 
    nano /opt/lan-orangutan/config/config.ini
# scan_targets = 192.168.XXX.0/24
  4. Stack in Portainer deployen

    Portainer → Stacks → Add Stack → Web Editor → Inhalt von docker-compose.yml einfügen → Deploy

  5. Web-UI aufrufen 
    http://<HOST-IP>:2910

⚙️ Konfiguration

ParameterStandardBeschreibung

scan_interval300Scan-Intervall in Sekunden
enable_port_scantruePort-Scan aktivieren
port_scan_range1-1024,3000-3099,8000-8099Zu scannende Ports

retention_days30Tage für Offline-Geräte-Historie
themeautoUI-Theme: light / dark / auto

🔧 Technische Hintergründe

ℹ️ network_mode: host – nmap benötigt direkten Zugriff auf das Netzwerk-Interface des Hosts für ARP-Scans (MAC-Adressen, Hersteller-Erkennung).
ℹ️ Port > 1024 – Der Container läuft als non-root User. Unter Linux dürfen nur root-Prozesse Ports unter 1024 binden.
ℹ️ cap_add: NET_RAW + NET_ADMIN – nmap braucht Raw-Sockets für ARP-Scans, ohne den Container komplett als root laufen zu lassen.

⚠️ Sicherheitshinweis

Die echte config.ini (mit deiner LAN-IP) niemals in ein öffentliches Repo einchecken. .gitignore schützt sie bereits. Für öffentlichen Zugriff einen Nginx Reverse Proxy mit Basic Auth vorschalten.

✅ Production-ready Docker Portainer nmap LAN Scanner Homelab

Production-ready setup for LAN Orangutan as a Portainer stack in your home network. Scans your LAN, discovers devices, shows open ports and MAC addresses – all via a clean web dashboard.

🚀 Step by Step

  1. Clone upstream repo & build image 
    git clone https://github.com/291-Group/LAN-Orangutan.git
cd LAN-Orangutan
docker build -t lan-orangutan:latest .
  2. Run setup script 
    git clone https://github.com/jbkunama1/hAI.LANOrangutan.as.a.Dockerstack.git
cd hAI.LANOrangutan.as.a.Dockerstack
chmod +x setup.sh && sudo ./setup.sh
  3. Set your subnet in config 
    nano /opt/lan-orangutan/config/config.ini
# scan_targets = 192.168.XXX.0/24
  4. Deploy stack in Portainer

    Portainer → Stacks → Add Stack → Web Editor → Paste docker-compose.yml → Deploy

  5. Open Web UI 
    http://<HOST-IP>:2910

⚙️ Configuration

ParameterDefaultDescription
port2910Web UI port (must be > 1024)
scan_interval300Scan interval in seconds
enable_port_scantrueEnable port scanning
port_scan_range1-1024,3000-3099,8000-8099Ports to scan
scan_targets192.168.XXX.0/24Target subnet
retention_days30Days to keep offline device history

🔧 Technical Background

ℹ️ network_mode: host – nmap needs direct access to the host’s network interface for ARP scans (MAC addresses, vendor detection).
ℹ️ Port > 1024 – The container runs as a non-root user. On Linux, only root processes can bind to ports below 1024.
ℹ️ cap_add: NET_RAW + NET_ADMIN – nmap needs raw sockets for ARP scans without running the entire container as root.

⚠️ Security Note

Never commit your real config.ini (containing your LAN IP) to a public repo. .gitignore already protects it. For public access, add an Nginx reverse proxy with Basic Auth in front.
